# WildBattleRoyale

Minecraft Java Edition의 일반 야생 월드에서 진행하는 Paper 개인전 배틀로얄 플러그인입니다.

참가자는 서로 떨어진 위치에서 시작해 바닐라 방식으로 자원을 모으고 장비를 제작합니다. 준비 시간이 끝나면 PvP가 활성화되고 월드보더가 줄어들며, 마지막까지 살아남은 플레이어가 승리합니다.

## 주요 기능

- 서버 접속 시 이동하는 보호된 하늘 로비
- 플레이어 간 거리를 고려한 무작위 시작 위치
- 준비 시간 동안 PvP 차단
- 시간에 따른 월드보더 축소
- 사망 및 경기 중 로그아웃 탈락 처리
- 탈락자 관전 모드와 재접속 처리
- 보스바와 사이드바를 통한 경기 정보 표시
- 마지막 생존자 자동 판정
- 경기 종료 후 플레이어 상태와 월드보더 복구
- 인원, 시간, 거리, 보더 및 로비 설정 지원

클래스, 커스텀 아이템, 보급 상자, 팀전 등의 부가 시스템 없이 야생 파밍과 최후의 전투에 집중합니다.

## 요구 환경

- Minecraft Java Edition 26.2
- Paper 26.2
- Java 25

현재 프로젝트는 Paper API `26.2.build.60-beta`를 기준으로 빌드됩니다.

## 설치

1. [Releases](https://github.com/yayokorea/paper-battle-royale/releases)에서 최신 jar 파일을 받거나 아래 방법으로 직접 빌드합니다.
2. jar 파일을 Paper 서버의 `plugins` 폴더에 넣습니다.
3. 서버를 시작합니다.
4. 처음 실행된 `plugins/WildBattleRoyale/config.yml`을 필요에 맞게 수정합니다.
5. 설정을 변경했다면 서버를 다시 시작합니다.

### 직접 빌드

Java 25 환경에서 실행합니다.

```bash
./gradlew clean build
```

완성된 파일은 `build/libs/WildBattleRoyale-<버전>.jar`에 생성됩니다.

## 빠른 시작

1. 서버에 접속하면 자동으로 하늘 로비로 이동합니다.
2. 참가자는 `/br join`을 입력합니다.
3. 최소 인원이 모이면 관리자가 `/br start`를 입력합니다.
4. 준비 시간 동안 자원을 모아 장비를 갖춥니다.
5. PvP가 활성화된 후 마지막 생존자가 될 때까지 전투합니다.

경기를 즉시 종료하려면 관리자가 `/br stop`을 사용할 수 있습니다.

## 명령어

| 명령어 | 설명 | 권한 |
|---|---|---|
| `/br join` | 다음 경기에 참가합니다. | `battleroyale.play` |
| `/br leave` | 시작 전에 참가를 취소합니다. | `battleroyale.play` |
| `/br start` | 모집된 플레이어로 경기를 시작합니다. | `battleroyale.admin` |
| `/br stop` | 진행 중인 경기를 종료하고 상태를 복구합니다. | `battleroyale.admin` |
| `/br status` | 경기 단계, 인원, 남은 시간과 로비 위치를 확인합니다. | 없음 |

`battleroyale.play`는 기본적으로 모든 플레이어에게 허용되며, `battleroyale.admin`은 OP에게 허용됩니다.

## 경기 진행

### 모집

플레이어는 하늘 로비에서 대기하며 `/br join`으로 참가합니다. 모든 참가자는 온라인 상태이고 설정된 로비 월드에 있어야 합니다.

### 준비

경기가 시작되면 참가자의 상태가 저장되고 인벤토리가 비워집니다. 각 참가자는 `0,0`을 중심으로 서로 충분히 떨어진 안전한 지상에 배치됩니다. 준비 시간에는 참가자 사이의 직접 공격과 투사체 피해가 차단됩니다.

### 전투

준비 시간이 끝나면 PvP가 활성화됩니다. 설정된 시점부터 월드보더가 최종 크기까지 줄어듭니다.

사망하거나 경기 도중 로그아웃하면 탈락합니다. 탈락자는 관전 모드로 전환되며, 마지막 생존자가 결정되면 경기가 종료됩니다. 최대 경기 시간이 지나도록 승자가 나오지 않으면 무승부로 종료됩니다.

### 종료와 복구

경기가 끝나면 참가자의 인벤토리, 방어구, 게임 모드, 체력, 허기, 경험치와 비행 상태를 복구합니다. 변경했던 월드보더와 스코어보드도 원래 상태로 되돌린 뒤 플레이어를 로비로 이동시킵니다.

## 경기 UI

경기 중 참가자와 탈락한 관전자에게 다음 정보가 표시됩니다.

- 보스바: 현재 단계, 남은 시간, 진행률
- 사이드바: 생존자 수, PvP 상태, 현재 보더 크기, 보더 상태, 경기 잔여 시간

UI는 1초마다 갱신되며 경기가 끝나면 자동으로 제거됩니다.

## 하늘 로비

플러그인은 `lobby-world`의 X/Z `0,0` 상공에 원형 로비를 생성합니다.

- Y=200부터 위와 아래 방향으로 빈 공간을 검색합니다.
- 로비가 들어갈 공간 전체가 비어 있을 때만 생성합니다.
- 기존 블록이나 건축물은 덮어쓰지 않습니다.
- 선택한 위치는 `plugins/WildBattleRoyale/lobby-location.yml`에 저장합니다.
- 블록 설치와 파괴, 폭발, 화재, 액체, 피해 및 허기 감소를 차단합니다.
- 로비에서 추락하면 중앙으로 이동합니다.

저장된 로비의 중앙 표식이 사라졌거나 구조가 변경된 경우 플러그인은 기존 블록을 덮어쓰지 않고 활성화를 중단합니다. 로비를 옮기려면 서버를 정지한 뒤 기존 로비와 `lobby-location.yml`을 직접 정리하십시오.

## 설정

설정 파일은 `plugins/WildBattleRoyale/config.yml`입니다.

### 경기 설정

| 항목 | 기본값 | 설명 |
|---|---:|---|
| `min-players` | `2` | 경기 시작에 필요한 최소 인원 |
| `max-players` | `12` | 참가 가능한 최대 인원 |
| `preparation-seconds` | `900` | PvP가 차단되는 준비 시간 |
| `max-match-seconds` | `3600` | 준비 시간을 포함한 최대 경기 시간 |
| `teleport-radius` | `1500` | `0,0` 기준 시작 위치 탐색 반경 |
| `minimum-player-distance` | `350` | 시작 위치 사이의 최소 거리 |
| `random-location-attempts` | `80` | 참가자당 안전 위치 탐색 횟수 |

### 월드보더 설정

| 항목 | 기본값 | 설명 |
|---|---:|---|
| `border-start-size` | `3500.0` | 경기 시작 시 보더 한 변의 길이 |
| `border-end-size` | `75.0` | 축소가 끝난 뒤 보더 한 변의 길이 |
| `border-shrink-delay-seconds` | `900` | 경기 시작 후 보더 축소까지의 시간 |
| `border-shrink-duration-seconds` | `2400` | 보더가 최종 크기에 도달하는 시간 |

### 로비 설정

| 항목 | 기본값 | 설명 |
|---|---:|---|
| `lobby-world` | `world` | 로비를 생성할 월드 이름 |
| `lobby-y` | `auto` | 자동 높이 탐색 또는 고정 Y 좌표 |
| `lobby-search-start-y` | `200` | 자동 높이 탐색을 시작할 Y 좌표 |
| `lobby-radius` | `8` | 원형 로비의 반지름 |
| `lobby-search-step` | `4` | 후보 높이 사이의 탐색 간격 |

`border-shrink-delay-seconds`는 준비 시간 이상이어야 하며, 최대 경기 시간은 준비 시간보다 길어야 합니다. `lobby-y`에 숫자를 지정하더라도 해당 공간에 기존 블록이 있으면 로비를 생성하지 않습니다.

## 운영 시 참고사항

- 안전 위치는 단단한 지면과 플레이어의 머리 공간을 검사하지만 인접한 용암, 구조물 내부 등 모든 위험 요소를 판단하지는 않습니다.
- 사망 시 아이템과 경험치 드롭은 서버의 바닐라 규칙을 따릅니다.
- 경기 중인 플레이어의 기존 스코어보드는 일시적으로 교체됩니다.
- 정상 종료, `/br stop`, 시작 오류 및 플러그인 비활성화 시 예약 작업과 월드보더를 정리합니다.
- 서버 프로세스가 강제로 종료되면 메모리에 저장된 참가자 상태를 복구하지 못할 수 있습니다.
- 별도 로비 월드, 팀전, 데이터베이스 및 다른 미니게임 플러그인은 필요하지 않습니다.

Pull Request에서는 Java 25 환경의 전체 빌드와 테스트가 자동으로 실행됩니다. 커밋과 PR 제목은 `feat:`, `fix:`, `docs:` 같은 [Conventional Commits](https://www.conventionalcommits.org/) 형식을 사용합니다.

`main`에 병합된 변경사항은 Release Please가 하나의 릴리스 PR에 모읍니다. 이 PR을 병합하면 버전, 변경 이력, Git 태그와 GitHub Release가 생성되고 빌드된 jar 파일과 SHA-256 체크섬이 첨부됩니다.
